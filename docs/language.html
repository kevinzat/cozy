Overview

Cozy uses a text-based language for writing propositions, expressions, and proof rules. The language is designed to be readable and close to standard mathematical notation while being easy to type.

Identifiers

There are two kinds of identifiers, distinguished by their first character:

KindStarts WithExamplesUsed For
Variables Lowercase letter x, n, my_var Expression variables, quantified variables, set variables
Predicates Uppercase letter P, Even, MyPred Propositional variables, predicate names

Both can contain letters, digits, and underscores after the first character.

Propositions

Propositions are the statements you prove or assume. They are built from constants, predicates, logical connectives, and quantifiers.

Constants

SyntaxMeaning
trueAlways-true proposition
falseAlways-false proposition (contradiction)

Logical Connectives

SyntaxNameExample
notNegationnot P
andConjunctionP and Q
orDisjunctionP or Q
->ImplicationP -> Q
<->Biconditional (iff)P <-> Q

Quantifiers

SyntaxNameExample
forall x, ... Universal quantifier forall x, P(x) -> Q(x)
exists x, ... Existential quantifier exists k, n = 2*k

Quantifiers bind as loosely as possible — forall x, P(x) and Q(x) means forall x, (P(x) and Q(x)).

Predicates

A predicate name by itself is a propositional variable:

P              -- propositional variable
Q              -- propositional variable

A predicate with arguments is a predicate application:

Even(n)        -- predicate applied to expression n
Divides(a, b)  -- predicate applied to expressions a and b
GCD(a, b, d)   -- predicate with three arguments

Arithmetic Expressions

Expressions represent integer values. They appear inside predicates and comparisons.

SyntaxOperationExample
+Additiona + b
-Subtraction / Negationa - b or -a
*Multiplication2*k
^Exponentiationn^2

Integer constants are non-negative: 0, 1, 42. Use - for negation: -1.

Function calls use lowercase names with parentheses:

div(a, b)      -- integer division
mod(a, b)      -- modular remainder
gcd(a, b)      -- greatest common divisor
f(x, y, z)     -- custom function

Comparisons

Comparisons between expressions produce propositions:

SyntaxMeaningExample
=Equalityn = 2*k
<Less than0 < n
<=Less or equal0 <= mod(a,b)

Operator Precedence

From tightest to loosest binding:

Use parentheses to override precedence: (P or Q) and R.

Expression Precedence

PrecedenceOperators

Writing Definitions

Definitions are entered in the Definitions field of the theorem panel, one per line. Each definition has the format:

Name: forall vars, Name(args) <-> body

The proposition must be a universally quantified biconditional. Examples:

Even: forall n, Even(n) <-> (exists k, n = 2*k)
Odd: forall n, Odd(n) <-> (exists k, n = 2*k+1)
Divides: forall a, forall b, Divides(a, b) <-> (exists k, b = k*a)

Writing Theorems

Theorems are entered in the Theorems field, one per line:

Name: forall vars, hypothesis -> conclusion

The proposition must be a universally quantified implication. Examples:

MultEqns: forall a, forall b, forall c, forall d, (a = b and c = d) -> (a*c = b*d)
DivCancel: forall a, forall b, forall c, (c*a = c*b and not (c = 0)) -> (a = b)

Line References in Rules

When writing forward rules, you reference earlier proof lines by number. Cozy supports two kinds of references:

Absolute References

Use dot-separated numbers to refer to any line in the proof:

ReferenceMeaning
1Line 1 of the main proof
3Line 3 of the main proof
3.2Line 2 of the subproof at line 3
3.2.1Line 1 of the sub-subproof at line 3.2

Relative References

Use ^ to refer to lines relative to the current proof level:

ReferenceMeaning
^1Line 1 of the parent proof
^^1Line 1 of the grandparent proof
Tip Relative references are useful in deeply nested subproofs when you need to reference a line from an enclosing proof without counting the full path.
